Business Rules
Financial Year Cycle — March to February. FY2025 = Mar 2025 → Feb 2026. The fy column stores the year the period starts (integer). Stored in parquet as 2024, 2025, 2026.
Sales Target — 30% growth rule target(month M, year Y) = sales(month M, year Y−1) × 1.30
Applied per rep AND company total. Fully automatic — no manual data entry at any point.
Growth Status Thresholds GREEN  ≥ 30%   AMBER  0 – 29%   RED  < 0% 
Authoritative Rep Assignment — smref (delivery file) is canonical. smno (on invoice) is secondary. Rep codes: BV = Bhadresh Vallabh · NP = Nikhil Panchal · AP = Amit Patel · AC = Aboo Cassim · BM = Byron Minnie
Boundary Deduplication — 2025-03-03 appears in both FY2024 and FY2025 source files. The FY2024 file is trimmed to dates strictly before 2025-03-03. The FY2025 version is kept as it has more columns.
Partial Month Detection — last data date before the 15th of the month. Shown as ⚡ Partial badge in the monthly tab dropdown. Currently applies to May 2026 (data through 4 May).
Revenue = Net of Credits — all figures are INVOICE ivnett minus CRNOTE ivnett. Credit notes are subtracted at the rep, account, and company level. Gross invoices and credit totals are shown separately on FY detail tabs.
How to Update the Dashboard
1
Drop the new YTD export — replace Pad/Sales_Invoices_YTD_2026.xlsx with the updated export from the accounting system. If a new financial year starts, add an entry to the FILES list at the top of consolidate_sales.py.
2
Rebuild the parquet — python consolidate_sales.py
Re-reads all 3 Excel files, re-applies both enrichment joins, overwrites Sales_Invoices_All.parquet. Takes ~10 seconds.
3
Rebuild and publish — python build_sales_dashboard.py
Recomputes all metrics, regenerates index.html, stages all changes with git add ., commits, and pushes to GitHub Pages automatically. Dashboard is live within ~60 seconds.
4
Update this page if anything structural changes — edit Sales Dashboard/architecture.html directly, then push:
git -C "Sales Dashboard" add architecture.html && git -C "Sales Dashboard" commit -m "update architecture" && git -C "Sales Dashboard" push
